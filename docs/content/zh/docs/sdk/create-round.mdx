# 创建投票

使用 SDK 通过 Registry 合约创建 AMACI 投票轮次。

## 最小示例

```typescript
import { MaciClient, MaciCircuitType } from '@dorafactory/maci-sdk';

const client = new MaciClient({ network: 'testnet' });

// 查询可用的 operator
const operators = await client.indexer.getOperators();
const operator = operators.find(op => op.status === 'Active');

// 创建投票
const round = await client.createAMaciRound({
  signer: wallet,
  operator: operator.address,

  startVoting: new Date(),
  endVoting: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),  // 7 天后

  title: '社区提案投票',
  voteOptionMap: ['选项 A', '选项 B', '选项 C'],

  circuitType: MaciCircuitType.QV,  // 二次方投票
  maxVoter: 100,

  // 投票权重分配：所有人获得 100 权重
  voiceCreditMode: { unified: { amount: '100' } },

  // 注册模式：静态白名单
  registrationMode: {
    sign_up_with_static_whitelist: {
      whitelist: {
        users: [
          { addr: 'dora1...' },
          { addr: 'dora1...' }
        ]
      }
    }
  },

  deactivateEnabled: false
});

console.log('合约地址:', round.contractAddress);
```

## 核心参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `signer` | Wallet | 签名钱包（轮次创建者） |
| `operator` | string | Operator 地址 |
| `startVoting` | Date | 投票开始时间 |
| `endVoting` | Date | 投票结束时间 |
| `title` | string | 投票标题 |
| `voteOptionMap` | string[] | 投票选项标签列表 |
| `circuitType` | MaciCircuitType | `QV`（二次方）或 `IP1V`（一人一票） |
| `maxVoter` | number | 最大投票人数 |
| `voiceCreditMode` | VoiceCreditMode | 投票权重分配方式（见下文） |
| `registrationMode` | RegistrationModeConfig | 注册方式与访问控制（见下文） |
| `deactivateEnabled` | boolean | 开启 deactivate → AddNewKey 流程（默认 `false`） |

## VoiceCreditMode（投票权重模式）

定义如何为参与者分配投票权重（voice credits）：

### Unified 模式

所有用户获得相同的固定投票权重：

```typescript
voiceCreditMode: { unified: { amount: '100' } }
```

### Dynamic 模式

每个用户的投票权重等于其 Oracle 签名证书中包含的金额。需与 `SignUpWithOracle` 注册模式配合使用：

```typescript
voiceCreditMode: 'dynamic'
```

## RegistrationModeConfig（注册模式）

统一枚举，控制谁可以注册以及如何注册。此字段取代了原来分散的 `whitelist`、`voiceCreditAmount`、`oracle_whitelist_pubkey`、`preDeactivateRoot` 和 `preDeactivateCoordinator` 字段。

### SignUpWithStaticWhitelist（静态白名单）

用户直接调用 `SignUp`，仅预定义列表中的地址可注册：

```typescript
registrationMode: {
  sign_up_with_static_whitelist: {
    whitelist: {
      users: [
        { addr: 'dora1abc...' },
        { addr: 'dora1def...' },
        { addr: 'dora1ghi...' }
      ]
    }
  }
}
```

`users` 中每条记录还支持可选的 `voice_credit_amount` 字段，可覆盖全局 `voiceCreditMode`，实现白名单内各用户差异化投票权：

```typescript
registrationMode: {
  sign_up_with_static_whitelist: {
    whitelist: {
      users: [
        { addr: 'dora1abc...', voice_credit_amount: '200' },  // 分配 200 积分
        { addr: 'dora1def...', voice_credit_amount: '100' },  // 分配 100 积分
        { addr: 'dora1ghi...' }                               // 回退到 voiceCreditMode 全局配置
      ]
    }
  }
}
```

### SignUpWithOracle（Oracle 认证）

用户调用 `SignUp` 时须提供后端签发的证书，无需固定地址列表：

```typescript
registrationMode: {
  sign_up_with_oracle: {
    oracle_pubkey: 'ORACLE_PUBLIC_KEY_HEX'
  }
}

// 用户注册时须提供证书：
await client.signup({
  signer: wallet,
  contractAddress: amaciAddress,
  maciKeypair,
  certificate: oracleCertificate,
  amount: '150'  // Dynamic voiceCreditMode 时必填
});
```

### PrePopulated（预填充匿名）

用户必须通过 `PreAddNewKey` 使用 ZK 证明注册，直接 `SignUp` 不可用。适用于参与者提前收到预生成密钥对的完全匿名场景：

```typescript
registrationMode: {
  pre_populated: {
    pre_deactivate_root: preDeactivateData.root,
    pre_deactivate_coordinator: {
      x: platformCoordPubkey[0].toString(),
      y: platformCoordPubkey[1].toString()
    }
  }
}
```

## Round 类型矩阵

`registrationMode`、`voiceCreditMode` 和 `deactivateEnabled` 三个参数相互独立，组合后决定 AMACI Round 的完整行为。理解它们的交互关系，才能为具体场景选择合适的配置。

### 配置速查表

| Round 类型 | `registrationMode` | `voiceCreditMode` | `deactivateEnabled` | 典型场景 |
|---|---|---|---|---|
| 标准白名单投票 | `SignUpWithStaticWhitelist` | `Unified` | `false` | DAO / 代币门控投票，固定成员列表，投票权相等 |
| 白名单 + 密钥轮换 | `SignUpWithStaticWhitelist` | `Unified` | `true` | 白名单成员注册后可通过 AddNewKey 匿名化身份 |
| Oracle 认证 + 固定积分 | `SignUpWithOracle` | `Unified` | `false` | 后端验证开放注册，所有投票者获得相同积分 |
| Oracle 认证 + 动态积分 | `SignUpWithOracle` | `Dynamic` | `false` | 代币/质押权重投票，每人积分由 Oracle 证书决定 |
| 预置密钥匿名投票 | `PrePopulated` | `Unified` | `false` | 完全匿名，平台预生成并分发密钥对，用户通过 ZK 证明注册 |

### `deactivateEnabled` 的工作原理

`deactivateEnabled` 是一个**可选的隐私升级开关**，可与 `SignUpWithStaticWhitelist` 或 `SignUpWithOracle` 组合使用，解锁两步式密钥轮换流程：

```
用户注册（链上可见）
  → 用户提交 DeactivateMessage（需支付 10 DORA）
  → Operator 处理 deactivate 批次
  → 用户使用 ZK 证明调用 AddNewKey（新身份，与原地址完全解绑）
```

经过此流程，Operator 能看到有新身份注册，但**无法将其追溯到原始地址**，为白名单注册用户提供链上匿名性。

```typescript
// 允许成员后续切换为匿名身份的 Round
await client.createAMaciRound({
  registrationMode: {
    sign_up_with_static_whitelist: {
      whitelist: { users: members.map(addr => ({ addr })) }
    }
  },
  voiceCreditMode: { unified: { amount: '100' } },
  deactivateEnabled: true,  // 开启 deactivate → AddNewKey 流程
  // ...
});
```

> **`PrePopulated` 与 `deactivateEnabled: true` 的区别**
>
> 两者都能实现身份匿名化，但发生时间不同：
> - `PrePopulated` — 匿名化发生在 Round **启动前**：平台预先构建 deactivate 树并分发密钥，用户从第一天起就以匿名方式注册。
> - `deactivateEnabled: true` — 匿名化发生在 Round **进行中**：用户先公开注册（身份可见），再通过 ZK 证明轮换到新密钥。
>
> 将 `PrePopulated` 与 `deactivateEnabled: true` 同时使用在技术上可行，但通常不必要。

### 参数组合有效性说明

| `registrationMode` | 可用 `voiceCreditMode: 'dynamic'`？ | `deactivateEnabled: true` 有意义？ |
|---|---|---|
| `SignUpWithStaticWhitelist` | 否（白名单中已可为每人单独设置积分） | 是 |
| `SignUpWithOracle` | 是（积分嵌入证书中） | 是 |
| `PrePopulated` | 否（基于 ZK 的注册，无 Oracle） | 通常不需要 |

## 投票模式

### 二次方投票 (QV)

票数的平方为消耗量，鼓励分散投票权：

```typescript
circuitType: MaciCircuitType.QV

// 用户有 100 voice credits
// 投 10 票消耗 100 credits (10²)
// 选项 A 投 5 票 + 选项 B 投 5 票 = 共消耗 50 credits
```

### 一人一票 (1P1V)

票数直接等于消耗量：

```typescript
circuitType: MaciCircuitType.IP1V

// 用户有 100 voice credits
// 投 100 票消耗 100 credits
```

## PrePopulated（匿名）模式

通过预先生成密钥对并分发给用户，创建完全匿名的投票轮次：

```typescript
import { OperatorClient, genKeypair } from '@dorafactory/maci-sdk';

const operatorClient = new OperatorClient({ network: 'testnet' });

// 1. 为所有参与者生成密钥对
const keypairs = Array.from({ length: 100 }, () => genKeypair());

// 2. 构建 pre-deactivate 树
const preDeactivateData = await operatorClient.buildPreDeactivateTree({
  coordinatorKeypair: platformCoordinatorKeypair,
  userKeypairs: keypairs
});

// 3. 以 PrePopulated 模式创建轮次
const round = await client.createAMaciRound({
  signer: wallet,
  operator: selectedOperator.address,
  title: '匿名投票',
  voteOptionMap: ['选项 A', '选项 B'],
  circuitType: MaciCircuitType.QV,
  maxVoter: 100,
  voiceCreditMode: { unified: { amount: '100' } },
  registrationMode: {
    pre_populated: {
      pre_deactivate_root: preDeactivateData.root,
      pre_deactivate_coordinator: {
        x: platformCoordinatorKeypair.pubKey[0].toString(),
        y: platformCoordinatorKeypair.pubKey[1].toString()
      }
    }
  },
  deactivateEnabled: false,
  startVoting: new Date(),
  endVoting: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
});

// 4. 通过安全渠道将密钥对分发给用户
// 每个用户收到：{ privateKey, publicKey, preDeactivateData }
```

用户收到密钥后可立即匿名注册，无需暴露真实身份。

## 费用

AMACI Round 涉及三种不同的费用，**付款主体各不相同**：

### 1. Round 创建费——由 Round 创建者支付

调用 `createAMaciRound` 时一次性支付，涵盖 Operator 处理整个 Round 的全部成本（消息处理、ZK 证明生成、计票及 gas）。金额根据 `maxVoter` 和电路类型自动计算。

```typescript
// 创建前可预先查询费用
const fee = await client.queryAMaciChargeFee({ maxVoter: 1000, maxOption: 10 });
console.log('创建费用:', fee);

// 创建 Round 时费用自动包含在交易中
const round = await client.createAMaciRound({ signer: wallet, /* ... */ });
```

### 2. 投票消息费——由每位投票者支付

每位投票者每次调用 `PublishMessage` 支付 **10 DORA**。由于 AMACI 将多条投票消息打包到单次调用中，这是**按调用次数计费**，而非按单条消息计费。

```typescript
// SDK 在调用 vote() 时自动处理此费用
await client.vote({
  signer: wallet,
  contractAddress,
  selectedOptions: [{ idx: 0, vc: 8 }],
  // ...
});
// 从投票者钱包扣除 10 DORA
```

### 3. Deactivate 消息费——由用户支付，仅在 `deactivateEnabled: true` 时适用

用户提交 `PublishDeactivateMessage` 以轮换身份时，每次调用支付 **10 DORA**。此费用仅在创建 Round 时设置了 `deactivateEnabled: true` 的情况下才会产生。

```typescript
// 仅在 deactivateEnabled: true 时适用
await client.deactivate({
  signer: wallet,
  contractAddress,
  maciKeypair
});
// 从用户钱包扣除 10 DORA
```

### 费用汇总

| 费用类型 | 金额 | 付款方 | 发生时机 |
|--------|------|--------|---------|
| Round 创建费 | 按 `maxVoter` + 电路类型计算 | Round 创建者 | 调用 `createAMaciRound` 时 |
| 投票消息费 | 10 DORA / 次 `PublishMessage` 调用 | 投票者 | 每次调用 `vote()` 时 |
| Deactivate 消息费 | 10 DORA / 次 `PublishDeactivateMessage` 调用 | 用户 | 每次调用 `deactivate()` 时（仅 `deactivateEnabled: true`） |

## 常见问题

### 如何修改投票配置？

投票创建后，在投票开始前且 `num_signups == 0` 时，可以调用 `UpdateRegistrationConfig` 更新 `voiceCreditMode`、`registrationMode` 或 `deactivateEnabled`。若需更大范围的修改，则需创建新的投票轮次。

### 可以添加更多白名单地址吗？

不可以。使用 `SignUpWithStaticWhitelist` 时，白名单在创建时固定。如需灵活的访问控制，请改用 `SignUpWithOracle`。

### 投票期可以延长吗？

不可以。投票开始和结束时间在创建时确定，无法更改。

### Operator 地址从哪里获取？

通过 `client.indexer.getOperators()` 查询，或访问 https://vota.dorafactory.org/operators。
