# 费用结构

MACI 协议采用两层收费模型：创建轮次时支付 **Base Fee**（一次性），每次投票时支付 **Vote Fee**（按票计费）。所有费用以 DORA 代币计价（1 DORA = 10¹⁸ peaka）。

> **定价基准：** DORA = $0.005 USD，协议收取 10% 服务费

---

## Base Fee（轮次创建费）

Base Fee 在调用 Registry 合约的 `CreateRound` 时一次性支付，金额取决于所选电路规格。

| 电路规格 | 最大投票人数 | 最大选项数 | Benchmark 成本 | +10% 协议费 | Base Fee |
|----------|------------|----------|--------------|------------|---------|
| **2-1-1-5** | ≤ 25 | ≤ 5 | $0.0202 | $0.0222 | **5 DORA** |
| **4-2-2-25** | ≤ 625 | ≤ 25 | $0.1214 | $0.1335 | **27 DORA** |
| **6-3-3-125** | ≤ 15,625 | ≤ 125 | $0.9413 | $1.0354 | **208 DORA** |
| **9-4-3-125** | ≤ 1,953,125 | ≤ 125 | *(benchmark 进行中)* | — | **TBD** |

### Base Fee 计算说明

```
Base Fee (DORA) = ⌈ Benchmark成本 × 1.1 ÷ $0.005 ⌉

示例（4-2-2-25）：
  $0.1213993 × 1.1 ÷ $0.005 = 26.71 → 向上取整 = 27 DORA
```

### Benchmark 组成

Benchmark 成本来源于 ZK 证明的链上验证 gas 费用：

```
Benchmark = (instantiate_gas + process_gas) × ETH_price + storage_cost × DORA_price
```

| 电路 | 计算过程 |
|------|---------|
| 2-1-1-5 | (0.00095228 + 0.00105665) × $10 + 0.02 × $0.005 = $0.0202 |
| 4-2-2-25 | (0.01064518 + 0.00142975) × $10 + 0.13 × $0.005 = $0.1214 |
| 6-3-3-125 | (0.09211125 + 0.00138915) × $10 + 1.25 × $0.005 = $0.9413 |

## Vote Fee（单票费用）

每次用户调用 AMACI 合约的 `PublishMessage` 时，需支付统一的 Vote Fee。

| 定价依据 | 金额 |
|---------|-----|
| 标准成本（4-2-2-25 / 6-3-3-125） | ~$0.00018 USD |
| 含 10% 协议费 | ~$0.0002 USD |
| **实际定价** | **$0.0003 USD = 0.06 DORA** |

**所有电路规格统一收取 0.06 DORA / 票**（= 6 × 10¹⁶ peaka）

### Vote Fee 计算依据

```
Vote Fee = $0.0003 ÷ $0.005 per DORA = 0.06 DORA

4-2-2-25 vote benchmark:
  0.00001771 × 1 × $10 + 0.0002016 × 1 × $0.005 = $0.000178 USD

6-3-3-125 vote benchmark:
  0.00001602 × 1 × $10 + 0.00008 × 1 × $0.005 = $0.000161 USD

标准 vote cost ≈ $0.00018，+10% ≈ $0.0002，定价 $0.0003（含溢价）
```

---

## 费用汇总

| 费用类型 | 触发时机 | 支付方 | 金额 |
|---------|---------|-------|-----|
| Base Fee | `CreateRound` | 轮次创建者 | 按电路规格，5 / 27 / 208 / TBD DORA |
| Vote Fee | `PublishMessage` | 投票者 | 0.06 DORA / 票（统一） |
| Deactivate Fee | `PublishDeactivateMessage` | 用户 | 10 DORA / 次 |

---

## 费用流向

```
创建轮次
  └── Base Fee ──→ AMACI 合约（锁定）
                      ├── 10% → fee_recipient（协议收益）
                      ├── operator_reward（扣除延迟惩罚后）
                      └── 剩余 → admin

投票
  └── Vote Fee ──→ AMACI 合约（锁定，随 Base Fee 一起结算）
```

> 协议通过 10% 的 fee_rate 从 Base Fee 总量中提取收益。Vote Fee 和 Deactivate Fee 同样进入合约资金池，最终按相同比例分配。
