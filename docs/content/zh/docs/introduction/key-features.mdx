# 核心特性

AMACI 的主要功能和优势。

## 身份匿名化

AMACI 最重要的创新。使用 add-new-key 机制，即使 operator 也无法确定投票者身份。

通过零知识证明创建新身份，断开与原钱包地址的关联。Operator 只知道"有人投了票"，但不知道具体是谁，无法进行针对性贿赂或报复。

提供三种注册方式：signup（快速但不匿名）、add-new-key（完全匿名需等待）、pre-add-new-key（匿名且即时）。根据隐私需求选择。

## 投票内容加密

所有投票消息用 operator 公钥加密上链。区块链观察者只能看到加密数据，无法知道谁投了什么。

使用 ECDH + Poseidon 加密。只有 operator 能解密消息内容，但 operator 被零知识证明约束，必须正确处理所有投票。

## 抗贿选机制

密钥更改功能让投票者无法向第三方证明最终投票。

即使接受贿赂用旧密钥投票，仍然可以秘密更改密钥重新投票。最终只有新投票有效，贿赂者无法验证。结合 add-new-key，贿赂者连贿赂对象都找不到。

## 可重复投票

投票者可以多次投票，最后一次有效。

防止强制投票：被强制投某选项后，可以秘密重新投票。也允许改变主意或纠正错误。通过 nonce 机制确保按顺序处理，旧投票被覆盖。

## 零知识证明验证

Operator 处理投票后必须生成 ZK 证明，证明正确处理了所有投票。

证明确保：处理了所有消息、没有篡改投票、正确更新状态树、正确统计结果。任何人都可以验证证明，确认结果正确。合约自动验证证明，验证失败则拒绝。

## 多种投票模式

支持不同的投票规则：

**一人一票 (1P1V)** - Voice credit 直接作为票数。适合简单的民主投票。

**二次方投票 (QV)** - 票数的平方是消耗量。鼓励分散投票，防止大户垄断。适合公共物品资助、优先级排序等场景。

## 灵活的白名单

多种方式限制投票参与者：

**地址列表** - 指定具体地址，只有列表中的地址可以 signup。适合已知用户群体。

##专业 Operator 网络

许可制的专业节点提供服务，用户无需自己运行 operator 软件。

Operator 负责处理消息、生成证明、提交结果。创建投票时选择 operator 并支付服务费。Operator 网络由 Dora Factory 管理，确保服务质量。

## TypeScript SDK

完整的 SDK 简化集成。

提供客户端、operator、voter 等多个模块。包含完整的类型定义和文档。支持所有主要功能：创建投票、用户注册、投票、查询等。
